# HTML Layout Review Checklist

这份清单用于 `html-layout-review` 执行时逐页核对，避免漏掉常见布局问题。

## 1. 审查前先确认

- 审查对象是不是**最终 HTML 输出**
- 判断基准是不是**固定 `1920x1080`**
- 当前页面是不是本来就用于**录屏 / 幻灯片**
- 如果是固定舞台页面，**不要**把“不是响应式”本身记成问题

## 2. 每页必查项

### 2.1 标题区

检查：

- 标题是否超出画布或容器
- 标题换行后是否压到主体区
- 标题与 eyebrow / meta chip 之间是否互相挤压
- 长标题是否出现裁切、缺字、尾部被吞

### 2.2 主体内容区

检查：

- 主卡片、图表、流程节点、时间线项是否有重叠
- 内容模块是否跑出容器
- 多列布局是否明显失衡
- 原本宣称是双栏 / 三栏的页面，是否实际退化成单列堆叠

### 2.3 侧栏 / 底栏 / 辅助面板

检查：

- 侧栏有没有被固定浮层压住
- 字幕面板、说明面板、信号卡片是否过挤
- 长字段是否把 chip、panel 撑坏
- 面板里是否已经出现明显截断提示或真实裁切

### 2.4 固定浮层

检查：

- 右上角 / 左上角标签是否侵入正文安全区
- 页码、进度条、提示条是否挡住关键信息
- 固定元素和页面顶部内容之间是否有足够安全距离

### 2.5 底部与边缘安全区

检查：

- 关键文字是否贴近四边
- 卡片阴影、高亮放大后是否被裁边
- 底部字幕 / 面板 / 卡片是否压到边缘

## 3. 按页面类型加查

### 3.1 封面 / Hero 页

重点看：

- 大标题是否超高
- 标题和副标题是否抢空间
- 留白是否被装饰层打碎

### 3.2 Overview 总览页

重点看：

- 多节点 / 多卡片是否因为数量过多而溢出
- 总览图是否还能保持清晰阅读路径
- 重点高亮后会不会挡住邻近元素

### 3.3 Detail 细讲页

重点看：

- detail 页声称的版式是否真的实现了
- `focus` 区、`context` 区、标题区之间是否层次清楚
- 当前重点项是否被上下文卡片挤压

### 3.4 Fallback 页

重点看：

- fallback 页面是否只是把长文案硬塞进一个容器
- 视觉锚点区和摘要区是否互相抢空间
- summary 页是否因为留白不足变得像普通正文页

## 4. 哪些情况应直接记问题

以下情况应直接进入问题列表：

- 文字已经被切掉
- 元素已经出画
- 固定浮层已经挡住正文
- 模块之间已经互相重叠
- 页面声称是双栏 / 三栏，但实际结构已经失配
- 页面内已经存在“折叠”“截断”类明示提示，且说明当前布局容纳不下内容

## 5. 哪些情况不要乱记

以下情况不要轻易记成问题：

- 页面是固定 `1920x1080` 舞台
- 代码里存在 `overflow: hidden`，但画面中没有真实裁切
- 只是“也许某些未来文案会更长”
- 纯审美分歧，例如“还可以更高级”“可以更有设计感”

## 6. 建议输出模板

### 6.1 通过

- **审查结论**：通过
- **问题列表**：无

### 6.2 存在问题

- **审查结论**：存在问题
- **问题列表**：
  - **场景 1 / major / 重叠**：右上角固定标签压住右栏首行内容
  - **场景 4 / critical / 裁切**：底部一行卡片超出画布下边界
  - **场景 7 / minor / 信息过挤**：右侧信号卡片区行数过密，阅读压力明显过高
