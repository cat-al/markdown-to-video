# HTML Layout Review

对已经生成完成的 HTML 幻灯片做**最终画面验收**。只报告问题，不负责修复页面。

## Overview

核心原则只有两条：

1. **以最终渲染结果为准**，不是以代码表面结构为准
2. **只检查固定 1920x1080 画布内的问题**，不要把“固定录屏舞台本身”误报成错误

## When to Use

在这些场景使用本 skill：

- 用户要求检查 HTML 页面有没有**错位、越界、截断、裁切、重叠**
- `markdown-to-html` 刚生成完页面，需要做一次交付前验收
- 需要确认页面在**1920x1080** 录屏画布内是否安全

不要在这些场景使用：

- 还没有生成 HTML 页面
- 用户要的是**直接修复**而不是审查
- 用户要的是审美点评、风格建议、或内容事实核对

## 产物目录约定

> 完整约定见 `docs/project-output-convention.md`。

本 skill 接收**项目目录路径**作为参数（如 `output/001-cognitive-awakening/`），审查 `<项目目录>/presentation.html`。

## 最小输入形式

优先接受以下任一种输入：

- **项目目录路径**（自动定位 `<项目目录>/presentation.html`）
- **本地 HTML 文件路径**
- **可打开的预览 URL**
- 如果上下文里刚刚生成了唯一的 HTML 输出，可以默认以该文件为审查对象

如果对象不明确，先确认要审查哪个 HTML，而不是同时点评多个候选页面。

## 审查基准

固定按以下标准执行：

- **视口**：`1920x1080`
- **缩放**：默认按 `100%` 理解，不用任意缩放后的窗口做结论
- **目标**：检查最终录屏 / 导出画面里是否有真实问题

### 红线

- **不要**把“页面是固定 1920x1080、不是响应式”本身当成问题
- **不要**只因为看到 `overflow: hidden` 就直接报错；只有出现真实裁切、截断或遮挡时才记录
- **不要**输出一长串“可能会”“也许会”的代码推测；只记录**已经可见**或**几乎可以确定会进入最终画面**的问题

## 执行流程

1. **确认审查对象**
   - 明确要检查的 HTML 文件或预览地址
2. **固定画布基准**
   - 以 `1920x1080` 的最终画面为唯一判断基准
3. **逐页检查**
   - 按场景 / slide 检查标题、主体区、侧栏、底部区、固定浮层、装饰层
4. **按问题类型归类**
   - 错位、越界、裁切、截断、重叠、安全边距不足、信息过挤
5. **输出结构化问题清单**
   - 只给结论和问题列表，不给修复方案

详细检查项见 `references/review-checklist.md`。

## 输出格式

始终使用下面这个格式：

### 审查结论

- **通过**
- **存在问题**

### 问题列表

每条问题都要包含：

- **场景 / 页面**
- **严重级别**：`critical` / `major` / `minor`
- **问题类型**
- **问题描述**

### 通过态示例

- **审查结论**：通过
- **问题列表**：无

### 问题态示例

- **场景 2 / major / 重叠**：主标题换成两行后压到了右侧图表
- **场景 4 / critical / 越界**：底部数字卡片超出画布下边界，最终录屏会被裁掉
- **场景 6 / minor / 安全边距不足**：左侧说明文字距离边缘过近

## 分级标准

- **`critical`**：已经影响最终交付，例如核心文字被截断、元素出画、关键内容被遮挡
- **`major`**：明显存在版式风险，例如模块重叠、标题压缩主体区、信息拥挤到影响阅读
- **`minor`**：暂不阻塞交付，但应提醒修整，例如局部边距偏紧、视觉压力过高

## Quick Reference

| 问题类型 | 典型表现 |
|---|---|
| 错位 | 模块位置异常、对齐关系塌掉 |
| 重叠 | 标题压住图表、浮层挡住正文 |
| 越界 | 卡片、文字、图形跑出 1920x1080 画布 |
| 裁切 / 截断 | 文字缺字、尾部被切、容器吃掉内容 |
| 安全边距不足 | 关键信息贴边过近 |
| 信息过挤 | 虽未出界，但页面已经明显拥堵 |

## Common Mistakes

| 错误做法 | 正确做法 |
|---|---|
| 把固定录屏画布误报成非响应式问题 | 只检查内容有没有跑出固定画布 |
| 只看 CSS 就下结论 | 先看最终渲染结果，再用代码辅助定位 |
| 输出很多“高概率风险” | 只记录已发生或几乎确定进入最终画面的故障 |
| 顺手给修复方案 | 本 skill 只输出问题清单 |
| 混入审美评论 | 只讨论交付级视觉健康问题 |
